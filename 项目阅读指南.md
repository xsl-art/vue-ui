# vue-ui 项目阅读指南

> 面向阅读源码、二次开发与面试准备的说明文档。  
> 项目定位：基于 Vue 3 + TypeScript 的轻量级组件库，支持全局注册、按需引入与 npm 发布形态。

---

## 1. 项目概览

| 项          | 说明                                                                                |
| ----------- | ----------------------------------------------------------------------------------- |
| 技术栈      | Vue 3 + TypeScript + Vite 8 + Sass + Vitest + VitePress                             |
| 包名 / 版本 | `vue-ui` `0.1.2`（`package.json`）                                                  |
| 产物        | `dist/es/index.mjs`、`dist/umd/index.umd.js`、`dist/style.css`、`dist/types/*.d.ts` |
| 运行方式    | 开发演示：`pnpm dev`；文档站：`pnpm docs:dev`                                       |
| 代码风格    | Composition API、`<script setup>`、组件名 `Vk*`、样式类名 `vk-*`                    |

**核心能力：**

- 基础组件：Button、Icon、Tag、Alert、Switch
- 表单：Input、Select、Form / FormItem（async-validator 校验）
- 反馈：Message（命令式 API）、Dialog（Teleport + 滚动锁定）
- 浮层：Tooltip（Popper.js）、Dropdown（基于 Tooltip）
- 布局：Collapse / CollapseItem
- 文档：VitePress + Demo 预览插件
- 测试：Vitest + jsdom + @vue/test-utils

## 3. 组件清单与关系

| 组件         | 注册名         | 说明                                                    |
| ------------ | -------------- | ------------------------------------------------------- |
| Button       | VkButton       | 类型、尺寸、loading、原生 button / link                 |
| Icon         | VkIcon         | Font Awesome 封装                                       |
| Input        | VkInput        | v-model、clearable、showPassword、与 FormItem 联动校验  |
| Select       | VkSelect       | 单选/多选、可筛选、键盘导航，依赖 Input + Tooltip + Tag |
| Form         | VkForm         | provide 表单上下文，收集 FormItem                       |
| FormItem     | VkFormItem     | inject 规则与 model，async-validator                    |
| Switch       | VkSwitch       | 开关 v-model                                            |
| Tag          | VkTag          | 标签、可关闭                                            |
| Alert        | VkAlert        | 提示条                                                  |
| Tooltip      | VkTooltip      | hover/click 触发，Popper 定位                           |
| Dropdown     | VkDropdown     | 菜单项，复用 Tooltip 浮层                               |
| Dialog       | VkDialog       | Teleport、modal、拖拽、ESC、body 滚动锁                 |
| Collapse     | VkCollapse     | 手风琴 / 多开，provide 子项状态                         |
| CollapseItem | VkCollapseItem | 面板项                                                  |
| Message      | VkMessage      | 组件 + `createMessage()` 函数式调用                     |

**组合关系（阅读重点）：**

```
Dropdown ──► Tooltip ──► Popper.js + useClickOutside
Select   ──► Input + Tooltip + Tag + RenderVNode
Input    ──► inject FormItem ──► 触发 validate(blur/input/change)
Form     ──► provide ──► FormItem[] ──► validate() 汇总
Message  ──► h() + render() ──► document.body + useZIndex
Dialog   ──► Teleport + scrollLock(引用计数) + useZIndex
```

## 4. 库入口与发布

### 4.1 入口 `src/index.ts`

- `install(app)`：遍历 `components` 数组，`app.component(component.name, component)` 全局注册。
- 默认导出 `{ install }`，支持 `app.use(VueUi)`。
- 命名导出各组件及 `createMessage`、`closeMessageAll`。
- `export type *` 导出各组件 Props / Emits 类型。
- 副作用：引入 `@/styles/index.scss`、注册 Font Awesome `fas` 图标集。

### 4.2 构建 `vite.config.ts`

- `build.lib`：`entry: src/index.ts`，输出 **ES**（`dist/es/index.mjs`）与 **UMD**（`dist/umd/index.umd.js`）。
- `rollupOptions.external: ['vue']`：Vue 作为 peerDependency，不打进 bundle。
- `cssFileName: 'style'` → 使用者需 `import 'vue-ui/style.css'`。

### 4.3 类型 `tsconfig.build.json`

- `vue-tsc` 仅 `emitDeclarationOnly`，输出到 `dist/types`。
- `pnpm build` = `build:lib` + `build:types`。

### 4.4 package.json exports

```json
".": { "import": "./dist/es/index.mjs", "require": "./dist/umd/index.umd.js" },
"./style.css": "./dist/style.css"
```

## 5. 架构要点（按模块）

### 5.1 表单系统（Form / FormItem / Input）

**数据流：**

1. `Form` 接收 `model`、`rules`，`provide(formContextKey, { addField, removeField, ...props })`。
2. `FormItem` 在 `onMounted` 时 `addField`，持有 `prop`、校验方法、`inputId` / `errorId`（无障碍）。
3. `FormItem.validate(trigger)` 用 `async-validator` 的 `Schema` 校验对应字段；失败写入 `validateStatus`。
4. `Form.validate()` 遍历所有 field，合并错误后 `reject` 或返回 `true`。
5. `Input` 通过 `inject(formItemContextKey)` 在 blur / input / change 时调用 `validate(trigger)`。

**设计模式：** 与 Element Plus 类似——父组件注册子项、命令式 `ref.validate()`，而非完全依赖模板事件冒泡。

### 5.2 浮层与定位（Tooltip / Dropdown）

- **Popper.js**：`isOpen` 为 true 时在 `flush: 'post'` 后 `createPopper(trigger, popper, options)`。
- **触发器**：`hover` 用 debounce 处理 `openDelay` / `closeDelay`，且 hover 时互斥取消（`openDebounce.cancel()`）。
- **click**：`useClickOutside` 监听 document，点击容器外关闭。
- **manual**：Dropdown / Select 通过 `ref` 调用 `show()` / `hide()`。
- **Dropdown**：`menuOptions` 的 `label` 可为 VNode，经 `RenderVNode` 渲染。

### 5.3 命令式 Message

`src/components/Message/methods.ts`：

1. `createMessage(props)` 创建 `div` 容器。
2. `h(MessageConstructor, { id, zIndex, onDestory })` + `render(vNode, container)`。
3. 将 DOM 挂到 `document.body`，实例存入 `shallowReactive(instances)`。
4. 多条消息通过 `getLastBottomOffset` 计算垂直堆叠偏移。
5. 销毁：`render(null, container)` + `container.remove()`，并从 instances 移除。

这是典型的 **imperative API + 脱离组件树** 实现，面试常问。

### 5.4 Dialog

- `Teleport to="body"`，默认 `appendToBody: true`。
- `v-model` 控制显隐；打开时 `nextZIndex()`，可选 `lockBodyScroll()`。
- `scrollLock.ts` 使用 **引用计数** `scrollLockCount`，支持多个 Dialog 嵌套打开。
- 锁定方式：`body position: fixed` + 记录 `scrollTop`，关闭时 `scrollTo` 恢复。
- `draggable`：header mousedown 计算边界 clamp 偏移，CSS 变量 `--vk-dialog-translate-x/y`。
- `destroyOnClose`：关闭动画结束后卸载 body 插槽内容。

### 5.5 Collapse

- `provide(collapseContextKey, { activeNames, handleClick })`。
- `accordion` 模式仅允许一项展开；`modelValue` 为 `NameType[]`，双向 `update:modelValue`。

### 5.6 全局 z-index（useZIndex）

- 模块级 `ref(0)`，每次 `nextZIndex()` 自增，返回 `initialValue + zIndex`。
- Dialog 默认 initial 2000；Message 创建时也会取 z-index，避免层叠错乱。

### 5.7 样式体系

- `_var.scss`：CSS 变量（主色、成功、圆角、字体等），组件 SCSS 引用 `var(--vk-*)`。
- 各组件 `style.scss` 由 `styles/index.scss` 统一 `@use` 引入。
- 类名 BEM 风格：`vk-button`、`vk-form-item__label`、`is-disabled` 状态类。

### 5.8 RenderVNode

- 小型 functional 组件：`return () => props.VNode`，用于 Select 自定义 `renderLabel`、Dropdown 菜单 label、Message 自定义内容。

## 7. 关键文件阅读顺序（建议）

1. `package.json` → `src/index.ts` → `vite.config.ts`
2. `src/styles/_var.scss` → 任选一个简单组件（如 `Button/Button.vue`）
3. `hooks/useZIndex.ts`、`hooks/useClickOutside.ts`
4. `components/Tooltip/Tooltip.vue`（浮层基础）
5. `components/Form/Form.vue` → `FormItem.vue` → `Input/Input.vue`（表单链路）
6. `components/Message/methods.ts` → `Message.vue`（命令式 API）
7. `components/Dialog/Dialog.vue` → `scrollLock.ts`
8. `components/Select/Select.vue`（复杂组合组件）
9. `src/tests/components/form.spec.ts`（理解测试写法）

## 8. 本地运行

```bash
cd vue-ui
pnpm install

# 库开发（空 App，主要用于调试打包）
pnpm dev

# 组件文档站
pnpm docs:dev

# 构建发布产物
pnpm build

# 单测
pnpm test
```

**在业务项目中验证本地包：**

```bash
pnpm build && pnpm pack
# 在业务项目：pnpm add ../vue-ui/vue-ui-0.1.2.tgz
```

入口需：`import VueUi from 'vue-ui'`、`import 'vue-ui/style.css'`、`app.use(VueUi)`。

## 9. 已知注意点 / 可改进点

- `src/App.vue` 为空，日常演示主要依赖 VitePress docs，而非 dev 入口页面。
- 目录名 `commen` 为拼写笔误，不影响功能，重构时可改为 `common`。
- Tooltip 内变量名 `triggrtNode` 为拼写笔误。
- `useClickOutside` 在 click 触发模式下，document 级监听可能与 Dropdown 内点击时序有关，复杂场景需留意事件冒泡。
- Select、Form 等组件体积较大，可按需拆 composable（如 `useSelectKeyboard`）降低单文件复杂度。
- 未配置 unplugin-vue-components 自动按需，业务侧需手动 import 或全量 `install`。

## 10. 与 Element Plus / Ant Design Vue 的边界

本项目是 **学习型 / 个人向** 组件库：实现常用交互与表单校验链路，未覆盖 Table、DatePicker、国际化、主题切换引擎、无障碍全套规范等。

面试时可强调：**重点在「库工程化（打包、类型、文档、测试）」+「典型组件模式（provide/inject、Teleport、命令式 API、Popper）」**，而非组件数量对标大厂 UI 库。
